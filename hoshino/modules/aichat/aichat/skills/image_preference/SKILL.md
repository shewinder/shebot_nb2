# 图片喜好学习 SKILL

## 何时激活

1. **用户发送图片**（无论是否说"点评"）
2. **用户反馈**: "喜欢/不喜欢/好看/👍/👎"
3. **查看画像**: "我的喜好"
4. **偏好声明**: "我喜欢XX"、"不喜欢XX"、"不要XX"

## 核心原则：做审美分析师，不做标签计数器

你的任务不是统计标签频率，而是**理解用户审美内核**——找出表面标签背后的统一审美逻辑。

**分析时问自己：**
- 这些偏好背后有什么共同点？是什么审美标准在驱动选择？
- 用户选图是基于角色特征、构图美学、还是 IP 归属感？
- 用户拒绝的东西之间有没有规律？（比如拒绝所有打破日系理想化审美的内容）

**反例**（标签计数，禁止）：
> 用户喜欢蓝发(8)、少女(7)、泳装(5)

**正例**（审美提炼，要求）：
> 用户偏好日系高品质插画中对少女角色身体美感的委婉、精致表达。浅色发、紫/蓝眸、泳装/贴身服饰等是常见加分项，但只有在它们服务于整体角色美感和构图时才有效；用户拒绝的是打破这套审美框架的内容（欧美风、福瑞、夸张肉体、粗暴高负载或低俗猎奇表达）。

## 判断优先级：整体审美和内容语义优先

更新画像时必须按以下顺序判断，不允许反过来用浅层标签覆盖整体观感：

1. **整体美感与完成度**：画面是否好看、精致、构图成立、角色是否有吸引力。
2. **内容语义与关系语义**：画面卖点是美学化表达、角色魅力和氛围，还是强制、粗暴、羞辱、猎奇、低俗、恐怖、非合意等高语义负载内容。
3. **角色设计与风格**：日系/二游角色设计、比例协调、表情气质、画风一致性。
4. **浅层特征**：发色、瞳色、服装、IP、单个道具或姿势。

**硬规则**：
- 发色、瞳色、服装、IP 只能作为弱到中等证据；不能单独支撑“强偏好”或高置信结论。
- 如果一张图整体不好看、完成度低、内容语义明显冲突，即使命中浅色发/泳装/女仆装，也只能记为“弱补偿”或“边界样本”。
- 如果一张图整体美感强、角色设计好，即使发色或服装不完全命中，也可以作为正向样本。
- “用户浏览过”不等于“用户喜欢图中所有元素”；尤其不能把高负载题材、特定角色元素、关系元素或行为标签直接提升为偏好。

## 题材与语义负载判断

带有强题材语义的图片不能只看角色外观标签。必须先判断画面的主要表达方式、关系语义和题材负载：

| 语义层级 | 判断方式 | 画像写法 |
|----------|----------|----------|
| 美学化/含蓄 | 构图、姿态、服饰、色彩、关系和氛围服务于整体美感 | 可正向记录 |
| 直白但不冲突 | 题材表达明显，但角色美感、构图或情绪仍是主体 | 记录为边界或中性 |
| 冲突/高负载/破坏审美 | 强制、羞辱、猎奇、低俗、粗暴、非合意、怪物化、极端身体表现、不适元素堆叠、恐怖恶心等成为主要卖点 | 降权或回避 |

**关于单一元素和硬回避**：
- 不要把用户对某种表现方式的负反馈，自动扩大成对所有同类标签的硬回避。
- 应区分：某元素是画面主体还是弱存在，是审美冲突的核心原因还是偶然伴随项。
- 只有用户明确表达“任何这类都不要/完全讨厌”或多次高权重负反馈稳定指向同一类内容时，才写成硬回避。
- 对 `pr查看` 中出现的争议元素，默认按低权重中性浏览处理，除非用户随后显式喜欢或不喜欢。

## 信号权重：不是所有信号都一样

| 来源 | 权重 | 含义 |
|------|------|------|
| 用户显式说"喜欢/好看" | ⭐⭐⭐ 高 | 明确正反馈 |
| 用户显式说"不喜欢/一般" | ⭐⭐⭐ 高 | 明确负反馈 |
| 用户直接文字声明偏好 | ⭐⭐⭐ 高 | 主动表达，最高信任 |
| 用户通过 pr 查看了图 | ⭐ 低 | 可能感兴趣，也可能是好奇或随机浏览 |
| 用户上传了图 | ⭐⭐ 中 | 主动行为，但不一定喜欢 |

**规则**：
- 低权重信号不能单独支撑一个 ⭐⭐⭐⭐⭐ 偏好条目
- 样本数 < 3 的偏好标记为 `🔶 初步信号（待确认）`，不要给高于 ⭐⭐⭐ 的评级
- 只有 ≥3 次显式正面反馈或持续出现 >5 次的模式才能标 `⭐ 已确认`

## 分析维度（8个核心维度）

分析每张图片时，从这8个维度输出：

| 维度 | 分析内容 | 示例值 |
|------|---------|--------|
| **1. 人物年龄** | 外表年龄特征 | 幼女/少女(15-18)/成女(20-30)/熟女 |
| **2. 身体特征** | 身材、体型、特征 | 兽耳、巨乳、眼镜、瘦弱、匀称、长腿、狐耳、猫耳 |
| **3. IP作品** | 是否同人及出处 | 原创、蔚蓝档案、原神、初音未来、FGO、东方 |
| **4. 完成度** | 作品完整程度 | 草稿、线稿、铺色、半成品、完成品、精致完成 |
| **5. 氛围** | 情感基调 | 治愈、忧郁、激昂、宁静、温馨、神秘、恐怖、浪漫 |
| **6. 人物关系** | 角色间关系 | 单人、情侣、姐妹、朋友、敌人、主仆、多人 |
| **7. 服装** | 穿着风格 | 校服、女仆装、和服、泳装、战斗服、日常服、JK |
| **8. 内容分级** | 暴露、亲密、暴力、恐怖等内容强度 | 0-全年龄、1-轻微、2-中度、3-重度 |

**附加维度**（可选）：
- 画风: 赛璐璐、厚涂、平涂、像素、3D渲染
- 色调: 暖色调、冷色调、高饱和、低饱和
- 质量: 1-10分
- 内容语义: 美学化/直白/粗暴/强制/羞辱/猎奇/低俗/恐怖/怪物化/不适元素堆叠/夸张表现

**跨维度综合（重要）**：不要孤立地看每个维度。必须尝试找出跨维度的组合模式。例如：
- "俯视+泳装+少女" → 对特定视角下少女身体美感的审美凝视
- "精致完成+赛璐璐+二游IP" → 受过日系商业插画审美训练
- "银白发+紫瞳+贴身裙装" → 对特定色彩搭配+身体线条展现的偏好

**禁止的综合方式**：
- "银白发+绿瞳+关系互动" → 直接判断为强匹配，而忽略画面实际是高语义负载表达。
- "泳装+臀部视角" → 直接判断为正向，而忽略画面是否粗暴、低完成度或破坏角色美感。
- "出现某元素" → 直接写入强回避，而不判断它是否是画面主体、是否构成审美冲突的核心原因。

## IP 偏好 ≠ 审美偏好

IP（作品出处）是内容筛选器，不是审美偏好本身。用户追的是角色视觉特征，不是品牌。

**规则**：
- 如果用户喜欢某 IP 的角色但对同画风非 IP 图偏中性 → 偏好在于角色设计特征，不是 IP
- IP 章节记录的是"哪些作品的角色设计符合用户审美"，不是"用户喜欢这个作品"
- 当角色特征和 IP 冲突时，以角色特征为准
- 当 IP/角色命中但画面表达粗暴、高负载或低完成度时，以画面表达为准，不要让 IP 抬高结论

## 场景 1: 用户发图片

**AI 执行**:
1. 多模态分析图片（必须输出8维度）
2. 简要点评（1-2句话总结主要特征）
3. `read_file` 读取现有画像
4. **跨维度综合**：这张图与已有偏好有什么关联？强化了什么模式？引入了什么新信号？
5. 按信号权重更新画像结论；`pr查看` 只允许轻微调整证据摘要或边界判断
6. 压缩旧历史，控制全文长度后用 `write_file` 写回一份完整但精炼的画像

**点评示例**:
```
🎨 赛博朋克兽耳少女，冷色调霓虹光效
👤 少女(17岁)、狐耳、巨乳
📊 精致完成、全年龄
```

## 场景 2: 用户反馈

**触发**: "喜欢/好看/不错/👍" 或 "不喜欢/一般/👎"

**AI 执行**:
1. 关联到最近分析的图片
2. `read_file` 读取画像
3. 作为高权重信号更新偏好、回避或边界判断
4. 将事件写入“最近事件”，并把旧事件压缩进“证据摘要”
5. 控制全文长度后用 `write_file` 写回

**回复示例**:
```
✅ 已记录你的反馈
```

## 场景 3: 查看画像

**触发**: "我的喜好"/"画像"/"偏好"/"图像偏好"/"喜欢什么"

**AI 执行**:
1. **必须**先调用 `read_file` 读取画像文件（路径: `aichat/preferences/{user_id}.md`）
2. **等待工具返回后再回复** —— 不要假设没有画像
3. 展示：核心审美画像（叙事段落）+ 已确认偏好 + 初步信号 + 回避内容

**注意**: 用户询问画像时，不要直接说"没有"，必须先调用工具确认

## 场景 4: 直接偏好声明（无需图片）

**触发**: 用户直接用文字声明喜好，无需发图片

**示例语句**:
- "我不喜欢某题材"
- "我喜欢兽耳"
- "不要恐怖图"
- "偏爱少女"
- "拒绝福瑞"

**AI 执行**:
1. 识别声明类型：偏好 / 回避
2. 提取关键词：标签、特征、题材等
3. `read_file` 读取现有画像
4. 更新对应章节（偏好或回避）
5. 控制全文长度后用 `write_file` 写回

**不处理的情况**:
- 模糊的："还行吧"、"一般"（询问具体指什么）
- 临时的："今天不想看"（非长期偏好）

## 画像文件格式

路径: `data/aichat/preferences/{user_id}.md`

**结构目标**：只使用这一份 md 文件，但它必须是“当前画像报告”，不是无限增长的流水账。`pixivrank` 会优先使用文件前 800-3000 字符做筛选，因此最重要的推荐逻辑必须放在最前面。

**长度控制**：
- 目标长度：8000-12000 字符
- 硬上限：15000 字符
- 如果超过上限，必须先压缩“证据摘要”和“最近事件”，再写回
- 禁止为了保留每条历史而牺牲前部画像质量

```markdown
# 用户画像 - {user_id}

> 最后更新: YYYY-MM-DD | 证据规模: N 条 | 当前置信度: 高/中/低

## 🎯 推荐摘要

（500-800 字，写给 pixivrank 和推荐模型看的最高密度摘要。必须覆盖：
整体审美内核、强匹配、降权/回避、边界条件。重要结论放在第一段，不要写历史流水账。）

示例:
> 强匹配：整体美感优先，偏好日系高完成度、角色可爱/精致、比例协调、构图有美学距离的二游式插画。
> 浅色发、紫/蓝眸、制服/泳装/女仆/丝袜等是常见加分项，但不能单独决定推荐；真正起作用的是这些元素
> 是否服务于角色美感和身体线条的美学化呈现。高语义负载内容要按表达方式判断：含蓄、优雅、服务角色魅力时可正向；
> 强制、羞辱、猎奇、低俗、怪物化、粗暴、不适元素堆叠、夸张表现等若成为画面卖点应强降权。
> 争议元素按语境判断：如果它是画面主体、破坏整体美感或承载粗暴高负载语义则降权；如果只是弱存在或偶然伴随项，不应直接写成硬回避。

## 🎯 核心审美画像

（一段话，回答"这个用户是什么样的人"。提炼表面标签背后的统一审美逻辑。
不是关键词列表，不出现标签(次数)。必须是一段完整的叙事分析。）

示例:
> 用户是一个受过日系商业插画审美训练的二游玩家。核心审美是高完成度、比例协调、
> 角色可爱/精致、构图有美学距离的日系理想化美感。浅色发、紫/蓝眸、泳装、女仆装、
> 贴身裙装等不是独立偏好，而是常见的加分载体：它们需要服务于角色魅力、身体线条和
> 画面完成度。对高语义负载内容的接受取决于它是否美学化、是否服务角色魅力；强制、羞辱、
> 猎奇、低俗、粗暴、怪物化、不适元素堆叠和夸张表现会破坏
> 审美距离，即使角色外观标签命中也应降权。

## 🎯 强匹配特征

"这意味着什么"，而不是仅仅列出标签。）

| 特征 | 权重 | 含义 |
|------|------|------|
| 日系高完成度 + 角色整体美感 | ⭐⭐⭐⭐⭐ | 最核心判断，画面必须首先好看、精致、比例协调 |
| 美学化身体线条表达 | ⭐⭐⭐⭐ | 服饰、构图、姿态服务于角色魅力，而不是粗暴直白表达 |
| 蓝发/银白发 + 紫/蓝眸少女 | ⭐⭐⭐ | 常见加分项；必须和整体美感、完成度共同出现才可提高推荐优先级 |
| ... | ... | ... |

## 🚫 降权与回避

（有明确负反馈、显式声明，或与核心审美持续冲突的内容，区分程度。不要把低权重浏览信号写成硬回避。）

### 强回避

- 福瑞/兽人（如果有明确负反馈或持续破坏核心审美）
- 欧美画风/美式粗线条
- ...

- 夸张 oversized 巨乳
```

## 写入原则

- 必须进行全量审美分析，但写回的是“精炼后的当前画像”，不是完整流水账。
- 每次更新都要维护 `推荐摘要`，确保最关键筛选逻辑出现在文件前 800 字内。
- `核心审美画像` 必须解释审美内核和整体观感，不要退化成标签计数。
- `强匹配特征` 只放稳定、高置信度、可用于推荐筛选的综合模式；发色、服装、IP 这类浅层标签通常只能作为加分项或边界条件。
- `边界与待观察` 用来保存低权重、混合信号、样本不足的判断。
- `证据摘要` 用主题归纳历史依据；当 `最近事件` 超过 15 条时，把旧事件合并进这里。
- `最近事件` 是短表格，只记录最近 10-15 条，不保存完整历史。
- 写回前必须检查长度；超过 15000 字符时先压缩，不能直接写回超长画像。
- 写回前必须检查是否把“用户不喜欢某个表现方式”误扩大成“所有同类标签都强回避”。负反馈通常指向具体表现方式、画面主体或语义冲突，不一定指向整个标签类别。

## 工具使用

```python
# 获取用户ID
user_id = session.user_id

# 读取画像（路径相对于 data 目录）
read_file(path=f"aichat/preferences/{user_id}.md")

# 写入画像（AI组织完整、精炼、受长度控制的 Markdown）
write_file(path=f"aichat/preferences/{user_id}.md", content="...")
```

## 注意事项

- **⛔ 禁止使用 write_memory / read_memory**——画像文件必须通过 `write_file` / `read_file` 操作，路径为 `aichat/preferences/{user_id}.md`
- 每次分析必须覆盖8个核心维度
- **核心审美画像必须是叙事段落，不是标签列表**
- 低信号不支撑高评级：样本<3 → 初步信号，评级≤⭐⭐⭐
- pr查看 ≠ 喜欢，只是低权重参考信号
- IP是内容筛选器，不是审美偏好
- 发色、瞳色、服装、IP 不是核心审美本身；它们必须服务于整体美感才有推荐价值
- 带有强题材语义的图必须先判断内容表达方式，强制、羞辱、猎奇、低俗、怪物化等语义优先于浅层外观标签
- 争议元素要写成语境化判断，不要默认扩大为整个标签类别的强回避
- 历史不能无限追加；旧事件必须压缩为证据摘要
- 推荐摘要必须服务 pixivrank 筛选，强匹配与回避内容要放在最前面
- 用户说"喜欢/不喜欢"时立即更新画像
