# AI Chat 插件

支持以 `#` 开头的消息触发AI对话，支持 session 管理、人格管理和多模态。

## 功能特性

- ✅ 以 `#` 开头的消息自动触发AI对话
- ✅ Session管理：每个用户/群组独立的对话历史
- ✅ 人格管理：支持用户人格、群组默认人格、全局默认人格
- ✅ 角色卡导入：支持从 PNG 图片导入角色卡（TavernAI / SillyTavern 格式）
- ✅ 多模态支持：支持图片识别（需模型支持）
- ✅ 支持持久化存储，重启后对话历史不丢失
- ✅ 支持Session超时自动清理
- ✅ 支持手动清除对话历史

## 配置说明

配置文件位置：`data/config/aichat.json`

```json
{
  "apis": [
    {
      "id": "kimi",
      "name": "Kimi",
      "api_base": "https://api.moonshot.cn/v1",
      "api_key": "your-api-key",
      "model": "moonshot-v1-8k",
      "supports_multimodal": true
    }
  ],
  "default_api_id": "kimi",
  "max_history": 10,
  "session_timeout": 3600,
  "max_tokens": 2000,
  "temperature": 0.7,
  "default_persona": "",
  "max_saved_personas": 10
}
```

### 配置项说明

- `apis`: API 配置列表，支持多个 API 配置
  - `id`: API 配置唯一标识
  - `name`: API 显示名称
  - `api_base`: AI API的基础URL
  - `api_key`: API密钥
  - `model`: 使用的模型名称
  - `supports_multimodal`: 是否支持多模态（图片识别）
- `default_api_id`: 默认使用的 API ID
- `max_history`: 每个session保存的最大历史消息数（对话轮数）
- `session_timeout`: Session超时时间（秒），0表示永不过期
- `max_tokens`: 最大生成token数
- `temperature`: 温度参数，控制回复的随机性（0-1）
- `default_persona`: 默认人格文本
- `max_saved_personas`: 每个用户最多可保存的人格数量

## 使用方法

### 基本使用

在群聊或私聊中发送以 `#` 开头的消息即可触发AI对话：

```
# 你好，介绍一下你自己
```

发送图片（需要模型支持多模态）：

```
# 描述这张图片
```

### 人格管理命令

#### 查看当前人格
```
查看人格 / 查看AI人格 / 当前人格
```

#### 设置用户人格
```
设置人格 <人格描述>
例如：设置人格 你是一个友好的助手
```

#### 清除人格
```
清除人格 / 清除AI人格
```

#### 保存人格
```
保存人格 <名称> <人格描述>
例如：保存人格 猫娘 你是一个可爱的猫娘，说话温柔，喜欢撒娇
```

#### 列出已保存的人格
```
列出人格 / 人格列表 / 已保存人格
```

#### 使用已保存的人格
```
使用人格 <名称> / 切换人格 <名称> / 应用人格 <名称>
例如：使用人格 猫娘
```

#### 删除已保存的人格
```
删除人格 <名称>
例如：删除人格 猫娘
```

### 角色卡导入（新功能）

支持从 **TavernAI / SillyTavern** 格式的 PNG 角色卡图片导入人格，支持批量导入多张角色卡。

PNG 角色卡的元数据中会包含角色的 JSON 设定信息。

#### 个人导入（所有用户可用）

导入的角色卡保存到个人人格列表：

```
导入角色卡 / 导入人格 / 加载角色卡
```

#### 全局导入（仅超级用户）

导入的角色卡保存为全局预设人格，所有用户可用：

```
导入全局角色卡 / 导入全局人格 / 加载全局角色卡
```

#### 使用方法

以下方式均可触发：

1. **直接发送图片+命令**：
   ```
   导入角色卡 [附带一张或多张 PNG 图片]
   ```

2. **回复图片消息**：
   ```
   回复包含 PNG 图片的消息并发送：导入角色卡
   ```

3. **引用消息中的图片**：
   ```
   引用包含 PNG 图片的消息并发送：导入角色卡
   ```

#### 批量导入

支持同时导入多张角色卡图片，命令会自动：
- 识别哪些图片是有效的角色卡
- 跳过普通图片（非角色卡）
- 批量保存所有有效角色卡
- 返回导入结果统计

### Web 端导入

在 Web 管理界面中也可以导入角色卡：

1. 进入「AI 聊天管理」→「角色卡导入」标签页
2. 输入用户QQ号
3. 选择导入类型（个人人格 / 全局预设人格）
4. 拖拽或点击上传 PNG 角色卡图片
5. 支持批量上传多张图片
6. 查看导入结果统计

#### 角色卡格式说明

支持的角色卡字段：
- `name` - 角色名称（必需）
- `description` - 角色描述/背景故事
- `personality` - 性格特点
- `scenario` - 场景设定
- `first_mes` - 首条消息（用于开场）
- `mes_example` - 示例对话（用于学习语气）
- `creator` - 作者
- `tags` - 标签

**支持的 JSON 格式：**

1. **标准格式**（字段在根级别）：
```json
{
  "name": "角色名",
  "description": "角色描述..."
}
```

2. **Chara Card V2 格式**（字段在 `data` 中）：
```json
{
  "spec": "chara_card_v2",
  "data": {
    "name": "角色名",
    "description": "角色描述..."
  }
}
```

## Session管理

- **群聊Session**: 每个用户在每个群组有独立的对话历史
- **私聊Session**: 每个用户有独立的私聊对话历史
- **自动清理**: 超过 `session_timeout` 时间的Session会自动清理
- **持久化**: Session数据保存在 `data/aichat/aichat_sessions.json`

## 人格存储

人格数据持久化保存在以下位置：
- `data/aichat/aichat_personas.json` - 当前使用的人格
- `data/aichat/aichat_saved_personas.json` - 用户保存的人格
- `data/aichat/aichat_global_presets.json` - 全局预设人格

## 支持的API

本插件支持所有OpenAI兼容的API，包括但不限于：

- OpenAI官方API
- Moonshot AI (Kimi)
- Azure OpenAI
- 其他OpenAI兼容的API服务

## 注意事项

1. 首次使用前需要在配置文件中设置 `api_key`
2. Session数据会持久化保存，重启后不会丢失
3. 建议根据API服务的限制合理设置 `max_tokens` 和 `max_history`
4. 人格设置会立即生效，但已存在的对话历史不会自动清除，可使用「清除对话」命令
