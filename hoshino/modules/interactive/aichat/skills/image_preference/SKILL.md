# 图片喜好学习 SKILL

## 何时激活

1. **用户发送图片**（无论是否说"点评"）
2. **用户反馈**: "喜欢/不喜欢/好看/👍/👎"
3. **查看画像**: "我的喜好"
4. **偏好声明**: "我喜欢XX"、"不喜欢XX"、"不要XX"

## 分析维度（8个核心维度）

分析每张图片时，必须从这8个维度输出：

| 维度 | 分析内容 | 示例值 |
|------|---------|--------|
| **1. 人物年龄** | 外表年龄特征 | 幼女/少女(15-18)/成女(20-30)/熟女 |
| **2. 身体特征** | 身材、体型、特征 | 兽耳、巨乳、眼镜、瘦弱、匀称、长腿、狐耳、猫耳 |
| **3. IP作品** | 是否同人及出处 | 原创、蔚蓝档案、原神、初音未来、FGO、东方 |
| **4. 完成度** | 作品完整程度 | 草稿、线稿、铺色、半成品、完成品、精致完成 |
| **5. 氛围** | 情感基调 | 治愈、忧郁、激昂、宁静、温馨、神秘、恐怖、浪漫 |
| **6. 人物关系** | 角色间关系 | 单人、情侣、姐妹、朋友、敌人、主仆、多人 |
| **7. 服装** | 穿着风格 | 校服、女仆装、和服、泳装、战斗服、日常服、JK |
| **8. NSFW等级** | 内容分级 | 0-全年龄、1-轻微、2-中度、3-重度 |

**附加维度**（可选）：
- 画风: 赛璐璐、厚涂、平涂、像素、3D渲染
- 色调: 暖色调、冷色调、高饱和、低饱和
- 质量: 1-10分

## 场景 1: 用户发图片

**AI 执行**:
1. 多模态分析图片（必须输出8维度）
2. 简要点评（1-2句话总结主要特征）
3. `read_file` 读取现有画像
4. 添加分析记录到历史表格
5. `write_file` 写回更新后的画像

**点评示例**:
```
🎨 赛博朋克兽耳少女，冷色调霓虹光效
👤 少女(17岁)、狐耳、巨乳
📊 精致完成、全年龄
```

## 场景 2: 用户反馈

**触发**: "喜欢/好看/不错/👍" 或 "不喜欢/一般/👎"

**AI 执行**:
1. 关联到最近分析的图片
2. `read_file` 读取画像
3. 在历史记录中添加 👍/👎 标记
4. 更新偏好总结（根据反馈调整权重）
5. `write_file` 写回

**回复示例**:
```
✅ 已记录你喜欢「赛博朋克+兽耳少女」的组合
```

## 场景 3: 查看画像

**触发**: "我的喜好"/"画像"/"偏好"

**AI 执行**:
- `read_file` 读取画像
- 展示：核心偏好摘要 + 高频标签 + 回避内容

## 场景 4: 直接偏好声明（无需图片）

**触发**: 用户直接用文字声明喜好，无需发图片

**示例语句**:
- "我不喜欢男同"
- "我喜欢兽耳"
- "不要恐怖图"
- "偏爱少女"
- "拒绝福瑞"

**AI 执行**:
1. 识别声明类型：偏好 / 回避
2. 提取关键词：标签、特征、题材等
3. `read_file` 读取现有画像
4. 更新对应章节（偏好或回避）
5. `write_file` 写回

**不处理的情况**:
- 模糊的："还行吧"、"一般"（询问具体指什么）
- 临时的："今天不想看"（非长期偏好）

路径: `data/aichat/preferences/{user_id}.md`

```markdown

> 基于 N 张图片分析

## 🎯 核心偏好摘要
- 最爱组合: XX + XX
- 关键词: XX(8次) XX(5次)

## 👤 角色偏好
- 年龄: 少女 > 成女
- 特征: 兽耳(8)、巨乳(5)、眼镜(3)

## 🎨 风格偏好
- 完成度: 精致完成
- 画风: 赛璐璐 > 厚涂

## 🏷️ 内容偏好
- IP: 原创(10)、蔚蓝档案(6)
- 氛围: 治愈(7)、温馨(5)
- 服装: 校服(6)、女仆装(4)

## 🚫 回避内容
- 恐怖、血腥
- 男性主角

## 📊 历史记录
| 时间 | 年龄 | 特征 | IP | 氛围 | 服装 | 反馈 |
|------|------|------|----|------|------|------|
| ... | ... | ... | ... | ... | ... | 👍/👎 |
```

## 工具使用

```python
# 读取画像
read_file(path="aichat/preferences/{user_id}.md")

# 写入画像（AI组织完整Markdown）
write_file(path="aichat/preferences/{user_id}.md", content="...")
```

## 注意事项

- 每次分析必须覆盖8个核心维度
- 历史表格至少包含：时间、关键标签、反馈
- 用户说"喜欢/不喜欢"时立即更新画像
